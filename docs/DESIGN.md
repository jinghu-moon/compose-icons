# compose-icons 文档入口

原 `DESIGN.md` 已拆分为两份正式文档：

| 序号 | 文档 | 用途 |
|------|------|------|
| 01 | [01_design_specification.md](./01_design_specification.md) | 设计边界、架构、API、发布与质量策略 |
| 02 | [02_implementation_tasks.md](./02_implementation_tasks.md) | 分阶段实施任务、验收标准、交付顺序 |

## 说明

- 本文件保留为入口页，避免外部链接直接失效。
- `01_design_specification.md` 负责回答“为什么这样设计、系统应当长什么样”。
- `02_implementation_tasks.md` 负责回答“按什么顺序实现、每一步完成标准是什么”。
- 设计范围已收敛为 `Android + Jetpack Compose + 静态单色/准单色 SVG icon pack`，不再笼统宣称“支持任意 SVG 图标源”。

## 推荐阅读顺序

1. 先阅读 [01_design_specification.md](./01_design_specification.md)。
2. 再按 [02_implementation_tasks.md](./02_implementation_tasks.md) 执行。
